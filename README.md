# NUC Legal AI

AI-powered Indian Supreme Court judgment analyzer with an integrated chatbot (Lincoln Lawyer).

---

## What it does

- Upload a judgment PDF or paste judgment text
- Extracts: parties, sections, outcome, category, case number
- ML prediction: Allowed / Dismissed with confidence score
- Similarity search against 400+ 2025 Supreme Court judgments
- AI correction of extracted fields via Cohere
- Chatbot (Lincoln Lawyer) powered by OCI Generative AI

---

## Requirements

- Python 3.11+
- An [OCI account](https://cloud.oracle.com) with Generative AI access (Chicago region)
- A [Cohere API key](https://dashboard.cohere.com) for the extraction correction pipeline

## Setup

### 1. Clone and create a virtual environment

```bash
git clone <repo-url>
cd legal_ai_project
python -m venv venv

# Windows
venv\Scripts\activate

# Mac/Linux
source venv/bin/activate
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

> Note: `torch` and `transformers` are large packages. The first install may take a few minutes.

### 3. Configure environment variables

Copy the example and fill in your credentials:

```bash
cp .env.example .env
```

Open `.env` and set:

```env
# Cohere — used for AI field correction on extracted judgments
COHERE_API_KEY=your_cohere_api_key
AI_CORRECTION_ENABLED=true

# Chatbot — OCI Generative AI
CHATBOT_ENABLED=true

OCI_USER_OCID=ocid1.user.oc1..xxxx
OCI_TENANCY_OCID=ocid1.tenancy.oc1..xxxx
OCI_FINGERPRINT=xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx
OCI_PRIVATE_KEY_PATH=C:/path/to/your/oci_private_key.pem
OCI_REGION=us-chicago-1
OCI_COMPARTMENT_ID=ocid1.compartment.oc1..xxxx
OCI_CHAT_MODEL_ID=cohere.command-a-03-2025
```

To get OCI credentials:
1. OCI Console → Identity & Security → Users → your user → API Keys → Add API Key
2. Download the `.pem` private key file
3. Copy the fingerprint shown
4. OCI Console → Identity → Compartments → copy your compartment OCID

### 4. Download the InLegalBERT model (for ML features)

Download the model files from [HuggingFace — law-ai/InLegalBERT](https://huggingface.co/law-ai/InLegalBERT/tree/main) and place them in:

```
legal_ai_project/models/InLegalBERT/
```

Required files: `config.json`, `tokenizer_config.json`, `vocab.txt`, `pytorch_model.bin`

> If this folder is missing, the app will auto-download from HuggingFace on first run (requires internet).

### 5. Run database migrations

```bash
python manage.py migrate
```

### 6. Build ML artefacts

This step computes embeddings and trains the outcome classifier.
Requires `data/processed.csv` to exist (included in the repo).

```bash
python manage.py build_ml
```

This creates:
- `data/embeddings.npy` — InLegalBERT embeddings for all 400 judgments
- `data/model.pkl` — trained Logistic Regression classifier

> This step takes 5-15 minutes depending on your hardware (CPU vs GPU).

### 7. Start the development server

```bash
python manage.py runserver
```

Open [http://127.0.0.1:8000](http://127.0.0.1:8000)

## Customizing AI Prompts

All chatbot prompts are in one file:

```
app/chatbot/prompt_config.py
```

Edit that file to change how the AI responds — tone, style, what it covers, disclaimers, etc.
No other files need to be touched for prompt changes.

## Troubleshooting

**`No module named 'oci'`** — run `pip install oci`

**`Chatbot unavailable — OCI credentials not configured`** — check all `OCI_*` values in `.env` and confirm the `.pem` file path is correct

**`Classifier model not found`** — run `python manage.py build_ml`

**Slow chatbot responses (15-25s)** — this is normal for OCI GenAI on-demand inference. The first request after server start is slowest (cold start).

**`No module named 'transformers'`** — run `pip install transformers torch`
